1. Abstract

This report describes the EsseR (Essential Resource Planner) Project and its feature, as well as details about its progress. For more tehnical details, refer to this document [1].

2. Introduction

This project is a Web Application that allows the management of resources of any type in an organization.

The users or the groups of users will be notified about any change regarding the users or products, via e-mail or in-app.

The data regarding the management will be exported in CSV, HTML , XML and PDF format.

5. Working method

The application was developed in two stages: the Front-End and the Back-End. The Front-End uses HTML, CSS and ECMA-Script, which implementation can be found at '/app/view/'. route. The public resources(like CSS classes and ECMA-Script functions) can be found at '/public/resources/'.. For more info, consult: [1].

The Back-End was designed based on the MVC Pattern, coded in PHP, without using any framweork. All the MVC components (models, views and controllers) are routed and classified in the '/app/'. For more info, consult: [1].

We used Oracle 11g version 2 for the database, and the php_oci8 to make the connection between the web-server and DB. As a particular note, we implemented for each type of Table an Adapter Mapper, in other to create a standard for DB commands (like select insert and update), to make a bridge for the triggers and procedures from PL/SQL and also for sequrity reasons.

For each functionality of our application, we implemented separate Views:

  • Admin Control Panel(AdminCP)
  • Home (User landing page)
  • Logs
  • Notifications
  • Products
  • Groups
  • Reports

The AdminCP component has a special role, giving the root-admin of the application the special rights to approove the new created user accounts, to delete user accounts, and to alter/modify all the entities existent from the database, using automated triggers for constraints re-routing.

The Home component offers a brief review about the management situation of the application, giving details about the users and products that have been modified or added.

The Logs component offers a detailed list about any event that occured on the users, products, and specific groups, in order to make an up-to-date situation about the last modification in the Database. Using automated triggers written in PL/SQL, it collects different types of logs and others them after the date when something was modified regarding groups, users or products.

The Notifications component offers details of great importance for the logged-in user, depeding on the ownership of the product-groups or the user groups he is part of. It will notify the users that manage that product group if the quantity of a specific product reached it's Warning Quantity Limit.

The Products component offers the core-functionality of our application, giving the user, based on its grade of hierarchy, options to create or remove product groups, if it's manager, or just to add or remove products and modify their quantity.

The Groups component offers, based on the grade of hierarchy of the user, the posibility to create User Groups that will manage specific Product Groups, and to add users to them, giving specific rights so as can manage group or can manage products, in case of an admin or a manager, or just to see the colleagues of user group, in case of a normal user.

The Reports component offers different situation of the products recent changes, which can be downloaded as different types of format.

As a Back-end logic of organisation, each of the previous listed component has it's own Controller than associates URI addresses to specific functions. Each has it's own render method, which will load the View, based on specific pre-defined or real-time conditions and situations in application.

Each time when the database had to be accesed for querrying, the Table had been accessed like $this->model('nameOfTheTable').

Then, for each loaded model of a specific table, we accesses specific methods stored in the afferent Mapper classes, like this: $this->model_class->get_mapper()->Command, where we can define arrays of properties to generate specific equivalent PL/SQL commands.

6. Development Platform

The usage of the Bitbucket - web-based version control repository hosting service, along with different local git clients like SourceTree and GitHub for Desktop allowed an organized developement of the application. Each member of the team worked on his branch, pushing commits in order to realise a specific functionality, merging with other branches in order to obtain core functionalities and merging into the master branch, where the main functional version of the application is stored. Any conflicts between the branches were solved by code-reviewing.

7. Install Method

This section describes the install steps, including the tools and local system settings and configuration.

System requirements: any Windows NT based OS.

First, we need the Apache HTTP Server, which can be found in the XAMPP suite of programs, which can be downloaded from the following link: LINK. Any Apache of PHP version greater than 7.2.5 will meet the requirements, but we strongly recommend to install the XAMPP for an easier first-configuration.

Open XAMPP -> Apache Module -> config, the select PHP (php.ini), or find the php.ini file associated with the Apache server.

Add the following two lines at the end of the file:

extension=php_oci8.dll

extension=php_oci8_11g.dll

Change the port of the application, using Config -> Apache (httpd.conf), by adding the following code, or searching for similar code, and chaging the value with a +2000 number. Ex:

Listen 2017

Now we need to install the dependencies regarding the Oracle DB

First, search for a Oracle Instant Client(x86 version). After finding a specific zip, like instantclient-basic-nt-12.2.x.x.x (version 12.2+ recommended), then unzip it anywhere you like, but remember its location.

After that, access System Properties, then Environment Variables. After that, add to the Path from System variables the path of the previous extracted folder. You should also see the \xampp\php path from the first step, if you installed XAMPP.

Then, you need to install Oracle Database Express Edition 11g Release 2 from the following link. NOTE: You must have an Oracle account in order to download.

Next, you must add in \xampp\php\ext folder the following files:

php_oci8.dll

php_oci8_11g.dll

php_oci8_12c.dll

You can acquire the previous files from php_oci8 archives, x86 bit version.

Now, you must download the master-branch from our repo: Link to repo. Copy the content of the archive.

Next, navigate to \xampp\htdocs and delete everything from that folder. Now, paste the content into the archive(the src folder) directly into htdocs.

Then, open from XAMPP, press Config then select Apache (httpd.conf). Search the DocumentRoot, then modify to DocumentRoot "X:/xampp/htdocs/src/public", where X is the drive where you installed XAMPP.

In the same file, modify <Directory> to <Directory "X:/xampp/htdocs/src/public">, where X is the drive where you installed XAMPP.
